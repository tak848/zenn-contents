# プロジェクト構成

## 組織方針

- 「記事/本 = 成果物」を中心に据え、関連する画像・設定を最小限の規約で揃える
- 新しいコンテンツは、既存のパターンに沿えば迷わず追加できる状態を維持する

## ディレクトリパターン

### 記事（`/articles/`）

**目的**: Zenn の記事（1ファイル = 1記事）  
**約束**: 先頭に YAML フロントマター（`title/emoji/type/topics/published` など）  
**例**: `articles/20251218-pdf-xml.md`

### 本（`/books/`）

**目的**: Zenn の本（複数章/設定をまとめて管理）  
**約束**: `zenn-cli` の生成物・配置ルールに従う  
**例**: `pnpm exec zenn new:book` で作成したディレクトリ配下

### 画像（`/images/`）

**目的**: 記事から参照する画像アセットの保管場所  
**配置**: 記事単位でサブディレクトリを切る（例: `images/<article-slug>/...`）  
**参照**: Markdown からは `/images/...` の絶対パスで参照する  
**例**: `![](/images/20241208-isucon14-writeup/top30.png)`

### メモ/下書き（`/z/`）

**目的**: 公開前提ではないメモ、構想、下書きの断片  
**約束**: 記事として公開する段階で `articles/` に移し、必要なら画像も整理する  
**例**: `z/opensearch.md`

## 命名規約（推奨）

- **記事ファイル**: `YYYYMMDD-<slug>.md`（kebab-case、時系列で並ぶ）  
  - 既存記事には例外もあるが、新規はこの形式を優先する
- **画像**: `images/<article-slug>/<meaningful-name>.(png|gif|...)`

## Zenn 記事（Markdown）の基本ルール

- 各記事は YAML フロントマターから開始し、キーと型を安定させる（文字列は引用符、配列は `[...]`）
- 画像は `/images/...` で参照し、記事と画像の対応が追える構造にする
- Zenn 独自記法（`:::message` / `:::details` など）は「読みやすさ」に効く範囲で使う

## 変更の指針

- ここに書かれたパターンに沿う変更であれば、ステアリングの更新は不要
- パターンを崩す新しい運用（例: 画像の置き方、命名の大変更）を導入する場合のみ更新する

---
_ディレクトリツリーの列挙ではなく、追加・運用の判断に効く規約を残します_
